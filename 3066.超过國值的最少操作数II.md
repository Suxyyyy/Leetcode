[[3066.超过國值的最少操作数II]](https://leetcode.cn/problems/minimum-operations-to-exceed-threshold-value-ii/description/)
```cpp
class Solution {
public:
    int minOperations(vector<int>& nums, int k) {
        priority_queue<long long,vector<long long>,greater<long long>>minHeap;
        int ans=0;
        for(int& x :nums) {
            minHeap.push(x);
        }
        while(minHeap.size()>=2 && minHeap.top()<k) {
            int top1=minHeap.top();
            minHeap.pop();
            int top2=minHeap.top();
            minHeap.pop();
            int mn=min(top1,top2);
            int mx=max(top1,top2);
            minHeap.push((long long)mn*2+mx);
            ++ans;
        }
        return ans;
    }
};
```
贪心策略：每次**合并当前数组中最小的两个元素**，能最快让数组的最小值增大，是达成条件的最优选择，无更优合并方式。

- 数组所有元素入最小堆，初始化操作次数 `ans=0`；
- 循环条件：`堆大小≥2 && 堆顶元素 < k`（堆顶是全局最小值，它达标则所有元素达标）；
- 循环内操作：弹出堆顶 2 个最小元素 → 按规则计算新值 → 新值入堆 → 操作次数 + 1；
- 不满足循环条件时，返回操作次数 `ans`。

原数组元素是 int，但计算 2*mn+mx 极易超出 int 范围，堆存储类型必须为 long long；

- 时间复杂度：O(nlogn+mlogn) ，n 为数组长度（建堆开销），m 为操作次数；
- 空间复杂度：O(n) ，堆存储所有元素。
