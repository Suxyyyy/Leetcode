[[455.分发饼干]](https://leetcode.cn/problems/assign-cookies/)
```cpp
class Solution {
public:
    int findContentChildren(vector<int>& g, vector<int>& s) {
        ranges::sort(g);
        ranges::sort(s);
        int n=g.size();
        int ans=0;
        int idx=s.size()-1;
        for(int i=n-1;i>-1;--i) {
            if(idx>-1 && s[idx]>=g[i]) {
                --idx;
                ++ans;
            }
        }
        return ans;
    }
};
```
### 核心思想：贪心算法 (Greedy)

为了满足尽可能多的孩子，我们应该**物尽其用**，避免浪费。

- **策略选择**：优先用“大饼干”满足“胃口大”的孩子；或者优先用“小饼干”满足“胃口小”的孩子。
    
- **局部最优**：每一次匹配都尝试消耗当前最能满足目标的资源。
    
- **全局最优**：达到满足孩子数量的最大化。

| **步骤**      | **操作**                               | **逻辑**                   |
| ----------- | ------------------------------------ | ------------------------ |
| **1. 排序**   | `ranges::sort(g)`, `ranges::sort(s)` | 贪心的前提是**有序**，以便按顺序比较。    |
| **2. 倒序遍历** | 从胃口最大的孩子 `g[i]` 开始看。                 | 最大的孩子如果不被满足，就没有饼干能满足他。   |
| **3. 尝试匹配** | 如果最大的饼干 `s[idx]` $\ge$ 最大的孩子 `g[i]`  | 匹配成功！饼干指针移动，计数增加。        |
| **4. 跳过**   | 如果饼干不够大。                             | 这个孩子太贪心，饼干满足不了，直接看下一个孩子。 |
